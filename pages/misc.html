A tale of Extended Iterable Unpacking in Python2

So the other day, i saw a question regarding how to unpack a tuple (1, 2, 3, 4, 5) so that a should get 1 and and b should get 2 and c should get the rest, which is 3, 4, 5

This can be done in python3 using the extended iterable unpacking(PEP-3132).

>>> a, b, *c = (1, 2, 3, 4, 5)
>>> a
1
>>> b
2
>>> c
[3, 4, 5]
>>>

But the problem was, they wanted to do this in Python2.

So for a fun, i commented:

>>> a, b, c = (lambda x, y, *z: (x, y, z))(*(1, 2, 3, 4, 5))
>>> a
1
>>> b
2
>>> c
(3, 4, 5)
>>>

But i never meant this to be used in real code. This was just for fun. Because readability matters. And now you have to explain this to your colleagues what exactly this does. And if it needs to be explained, then probably there are other better ways to write this.
